# Style specimens

A small set of `getdesign.md` examples.

Use this file to sharpen taste with concrete style systems.
Do not use it to clone websites.

## How to use

1. Find the main weakness in the live UI.
2. Read one matching specimen.
3. Take the principle, not the branding.
4. Apply one small fix.
5. Check the screenshot again.

Do not copy:
- brand colors
- gradients
- custom fonts
- hero layouts
- custom component geometry

## Specimens

### Linear
Use when the UI feels busy or sloppy.

Learn from it:
- quiet hierarchy
- precise spacing
- narrow widths
- low-noise chrome

Do not copy:
- the purple accent
- its exact control styling

### Vercel
Use when the UI needs more precision and less decoration.

Learn from it:
- contrast discipline
- whitespace
- clean composition

Do not copy:
- the exact monochrome mood
- over-sparse styling

### Notion
Use when the UI feels harsh or sterile.

Learn from it:
- warm neutrals
- editorial calm
- readable block structure

Do not copy:
- notebook aesthetics
- softness that kills hierarchy

### Mintlify
Use when docs or settings are hard to read.

Learn from it:
- readable type rhythm
- good content density
- restrained accents

Do not copy:
- docs-site chrome
- its green accent

### Supabase
Use when a dark UI feels muddy or loud.

Learn from it:
- one controlled accent
- better dark surface hierarchy
- cleaner product polish

Do not copy:
- its exact emerald identity
- glassy dark treatments

### Stripe
Use when marketing, auth, or pricing needs better hierarchy.

Learn from it:
- spacing
- section rhythm
- polished composition

Do not copy:
- gradients as a substitute for structure
- marketing moves inside dense app UI

## Quick matching guide

- too busy -> Linear
- too decorative -> Vercel
- too harsh -> Notion
- hard to read -> Mintlify
- dark and muddy -> Supabase
- weak polish in landing/auth/pricing -> Stripe

## Guardrail

If the patch starts looking more branded or decorative than the original system, back out.
The goal is better taste, not cosplay.
