# Taste reference

## Primary external reference

- **GStack design-review**
  - https://github.com/garrytan/gstack/tree/main/design-review

## What to learn from it

Use GStack design-review as a taste reference for:
- restraint
- spacing rhythm
- hierarchy
- cleaner overlays and panels
- making UI feel deliberate instead of patched together

## What not to do

Do **not** treat it as a license to redesign every component.
If the target UI uses a design-system component, keep that component native first.

Use GStack to improve:
- proportion
- width/spacing judgment
- placement judgment
- visual calm

Do not use it to justify unnecessary overrides of:
- radius
- internal layout
- component geometry
- shadow language
- typography scale

## Recommended pairing

Use this reference together with:
- `design-system-preserve` for native component guardrails
- `ui-review-loop` for screenshot-based critique and one-axis fixes
